# ProxyBar 1.0 GitHub And Homebrew Cask Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce a repeatable ProxyBar 1.0 release flow that publishes a GitHub Release asset and installs cleanly through a Homebrew Cask tap.

**Architecture:** Keep ProxyBar as a Swift Package app bundle built by `scripts/package-app.sh`, then add a deterministic zip artifact and checksum for GitHub Releases. Maintain a separate Homebrew tap repository with one cask file that points at the GitHub release zip and installs `ProxyBar.app`.

**Tech Stack:** Swift Package Manager, macOS app bundles, GitHub Releases, GitHub CLI, Homebrew Cask Ruby DSL.

---

## Release Decisions

- Release version: `1.0.0`
- Git tag: `v1.0.0`

- Release asset: `ProxyBar-1.0.0.zip`
- GitHub app repository: `baha2046/ProxyBar`
- Homebrew tap repository: `baha2046/homebrew-proxybar`
- Cask token: `proxybar`
- Install command after publishing: `brew install --cask baha2046/proxybar/proxybar`

### Task 1: Make Packaging Versioned And Cask-Friendly

**Files:**
- Modify: `scripts/package-app.sh`
- Modify: `.gitignore`

- [ ] **Step 1: Replace `scripts/package-app.sh` with a versioned release packager**

```bash
#!/usr/bin/env bash
set -euo pipefail

VERSION="${1:-1.0.0}"
BUILD_NUMBER="${BUILD_NUMBER:-1}"

ROOT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
APP_DIR="$ROOT_DIR/.build/ProxyBar.app"
CONTENTS_DIR="$APP_DIR/Contents"
MACOS_DIR="$CONTENTS_DIR/MacOS"
RESOURCES_DIR="$CONTENTS_DIR/Resources"
DIST_DIR="$ROOT_DIR/dist"
ZIP_PATH="$DIST_DIR/ProxyBar-$VERSION.zip"

cd "$ROOT_DIR"
swift build -c release --product ProxyBar

rm -rf "$APP_DIR" "$ZIP_PATH"
mkdir -p "$MACOS_DIR" "$RESOURCES_DIR" "$DIST_DIR"
cp "$ROOT_DIR/.build/release/ProxyBar" "$MACOS_DIR/ProxyBar"
swift run -c release IconGenerator "$RESOURCES_DIR/AppIcon.icns"

cat > "$CONTENTS_DIR/Info.plist" <<PLIST
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>CFBundleExecutable</key>
    <string>ProxyBar</string>
    <key>CFBundleIdentifier</key>
    <string>com.ericchan.ProxyBar</string>
    <key>CFBundleName</key>
    <string>ProxyBar</string>
    <key>CFBundleIconFile</key>
    <string>AppIcon</string>
    <key>CFBundlePackageType</key>
    <string>APPL</string>
    <key>CFBundleShortVersionString</key>
    <string>$VERSION</string>
    <key>CFBundleVersion</key>
    <string>$BUILD_NUMBER</string>
    <key>LSMinimumSystemVersion</key>
    <string>13.0</string>
    <key>LSUIElement</key>
    <true/>
</dict>
</plist>
PLIST

/usr/bin/ditto -c -k --keepParent "$APP_DIR" "$ZIP_PATH"

echo "Created $APP_DIR"
echo "Created $ZIP_PATH"
echo "SHA-256:"
/usr/bin/shasum -a 256 "$ZIP_PATH"
```

- [ ] **Step 2: Add the cask file**

Run this from the tap repo:

```bash
SHA256="$(shasum -a 256 /Users/ericchan/Documents/ProxyBar/dist/ProxyBar-1.0.0.zip | awk '{print $1}')"
cat > Casks/proxybar.rb <<RUBY
cask "proxybar" do
  version "1.0.0"
  sha256 "$SHA256"

  url "https://github.com/baha2046/ProxyBar/releases/download/v#{version}/ProxyBar-#{version}.zip"
  name "ProxyBar"
  desc "Menu bar app for WireGuard split tunneling"
  homepage "https://github.com/baha2046/ProxyBar"

  depends_on macos: :ventura

  app "ProxyBar.app"

  postflight do
    system_command "/usr/bin/xattr",
                   args: ["-dr", "com.apple.quarantine", "#{appdir}/ProxyBar.app"]
  end
end
RUBY
```

## Notes On Signing And Notarization

The plan above is enough for a personal GitHub Release and a personal Homebrew tap. For the smoothest macOS first-launch experience, add Developer ID signing and notarization before zipping:

```bash
scripts/package-app.sh 1.0.0
SIGNING_IDENTITY="$(security find-identity -v -p codesigning | awk -F '"' '/Developer ID Application/ {print $2; exit}')"
NOTARY_PROFILE="notarytool-profile"
codesign --force --deep --options runtime --sign "$SIGNING_IDENTITY" .build/ProxyBar.app
/usr/bin/ditto -c -k --keepParent .build/ProxyBar.app dist/ProxyBar-1.0.0-notary.zip
xcrun notarytool submit dist/ProxyBar-1.0.0-notary.zip --keychain-profile "$NOTARY_PROFILE" --wait
xcrun stapler staple .build/ProxyBar.app
rm dist/ProxyBar-1.0.0.zip dist/ProxyBar-1.0.0-notary.zip
/usr/bin/ditto -c -k --keepParent .build/ProxyBar.app dist/ProxyBar-1.0.0.zip
shasum -a 256 dist/ProxyBar-1.0.0.zip
```

If notarization is added, sign and staple before creating the final zip, then recompute the SHA-256 checksum and update the cask.
